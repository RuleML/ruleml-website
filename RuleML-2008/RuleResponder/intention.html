RuleML-2008 Intention

  • ARCHITECTURE
  • DESCRIPTION
  • INTENTION
  • ONTOLOGY

     

The Intentions of this use case are as follows:

  • Applies rule-based technologies to the project management in virtual organizations and collaborative teams
  • Merits the ideas of multi-agent systems, rule management systems, and service oriented and event driven architectures
  • Applies negotiation and distributed coordination mechanisms of rule-based complex event processing
  • Demonstrates the interoperation of different distributed platform-specific rule execution environment based on Reaction RuleML as a platform-independent rule interchange format transported over an enterprise service bus as transport middleware
  • Demonstrates the integration and interoperation of rule standards (RuleML) and Semantic Web (RDF, RDFS, OWL) and meta data standards (e.g. iCal, vCard)
  • Extends the Semantic Web with a pragmatic rule-based layer (Pragmatic Web), which defines the rules for using the information resources and ontologies to support human agents in their decisions and react autonomously as automated agents
  • Addresses real-world software engineering needs for a highly distributed, open, interoperable, efficient and scalable technical infrastructure
